# Clima e SLA

## Condições Climáticas

O sistema consulta diariamente informações climáticas por região.

Essas informações podem ser usadas para:

- apoiar planejamento de voo
- apoiar planejamento de campo
- enriquecer análise de risco
- priorizar regiões com maior risco

Exemplos de dados climáticos:

- temperatura
- umidade
- chuva
- vento

Esses dados podem ser armazenados historicamente por:

- região
- data

## SLA

O sistema possui regras de SLA (Service Level Agreement).

O SLA representa o tempo esperado para atendimento e resolução de problemas identificados.

Exemplos de regras:

- tempo máximo para validação de problema
- tempo máximo para aplicação de plano de ação
- alerta para itens vencidos

O sistema deve permitir:

- definir prazos por tipo de problema
- definir prazos por prioridade
- identificar itens vencidos
- registrar datas de criação, atendimento e conclusão
